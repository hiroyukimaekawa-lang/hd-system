# HD事業部 業務システム・ツール統合リポジトリ (hd-system)

本リポジトリは、HD事業部で運用している業務効率化システム、Chrome拡張機能、GAS (Google Apps Script)、データ連携・正規化ツールを一元管理するための社内専用リポジトリです。

---

## ディレクトリ構成

```text
hd-system/
├── README.md               # 本ドキュメント
├── docs/                   # 業務ツールポータルのWebページ (GitHub Pages等で公開)
│   ├── index.html          # ポータルメインHTML
│   ├── css/
│   │   └── style.css       # スタイル定義
│   ├── js/
│   │   └── main.js         # ポータルの検索・動的フィルタ・モーダル制御JS
│   ├── data/
│   │   ├── features.json   # 登録機能データソース
│   │   └── updates.json    # アップデート履歴データソース
│   └── assets/             # ポータル用静的アセット
├── extensions/             # Chrome拡張機能モジュール
│   ├── google-maps-scraper/      # Googleマップ店舗データ取得拡張機能 (v4.0)
│   └── restaurant-data-scraper/  # 食べログ・ホットペッパー店舗データ取得拡張機能

│   └── list-normalizer/    # CSV統合・重複排除・チェーン除外・CSV生成GAS

│   └── requirements.md     # 業務ツールポータルの要件定義書
└── src/                    # 共通ソースコード・その他のツール類
```

## 🚀 業務ツールポータル (`docs/`) の使い方・ローカル実行

本ポータルは **「非エンジニアでも直感的に使える」** 業務ツールの窓口（案内ポータル）です。

### 1. ローカルでの動作確認
HTML/JS/CSSとJSONファイルで構成されています。セキュリティ制約（CORS）のため、ブラウザで `index.html` をダブルクリックするだけではJSONが読み込めない場合があります。以下の方法で簡易Webサーバーを起動して閲覧してください。

#### VS Code (Live Server) を使用する場合:
- VS Codeで `docs` フォルダを開きます。
- 右下の「Go Live」ボタンを押すと、自動的にブラウザでローカルサーバー (`http://127.0.0.1:5500`) が立ち上がります。

#### Python を使用する場合 (ターミナル):
`docs` ディレクトリに移動し、以下のコマンドを実行します。
```bash
python3 -m http.server 8000
```
起動後、ブラウザで `http://localhost:8000` にアクセスしてください。

### 2. ツールの追加・更新方法
新しいツールをポータルに追加する場合、ソースコードを変更する必要はありません。
`docs/data/features.json` に新しいツールのオブジェクト（スキーマ定義は `requirements.md` 参照）を1件追加するだけで、ポータル上部のサマリー件数およびカード一覧、詳細モーダルが動的に自動生成されます。

また、システムを改修した際は、`docs/data/updates.json` に更新内容を追記することで、ポータル最下部の「システム更新・改善履歴」に自動反映されます。

## 🔒 セキュリティと機密情報取り扱いガイドライン

本リポジトリはパブリック（または複数の関係者がアクセス可能）な状態で運用されることを前提としているため、**機密情報の漏洩防止を徹底してください。**

1. **APIキー・認証トークンのハードコード禁止**
   - SerpAPIやその他の外部サービスAPIキーは、絶対にコードに直接記述してコミットしないでください。
   - GASでAPIを使用する場合は、スクリプトプロパティ (`PropertiesService`) を使用し、キー値はスプレッドシート管理画面から個別に設定します。
2. **営業リスト・顧客データ等の除外**
   - 営業先店舗リスト、電話番号入りCSV/XLSX、架電ログなどの個人情報・営業秘密データは、`.gitignore` によって自動的に除外される設定となっています。
   - ローカルでのテスト用CSVや実データを誤って `git add` しないよう、コミット前に `git status` で確認を行ってください。
3. **外部公開時の注意**
   - ポータルおよびリポジトリのURLはHD事業部の業務プロセスが記載されています。事業部外や外部の第三者へのURL共有は原則禁止とします。
