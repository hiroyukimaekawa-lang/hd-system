# CSV統合・正規化・取得漏れ分析・除外理由管理 GAS

本ディレクトリには、HD事業部の営業リスト作成業務でスプレッドシート上に実装・運用しているGoogle Apps Script (GAS) の機能概要と、コードテンプレートを配置します。

## 主な機能

### 1. CSV統合・正規化機能
- Googleマップ、食べログ、ホットペッパーから取得したCSVデータをスプレッドシートに一括取込
- 追加列 `検索ジャンル / 取得ステータス / 除外理由 / 詳細取得リトライ回数 / 一覧取得順` がある場合は分析に利用
- 店名、電話番号、住所、ジャンルを正規化し、`01_NORMALIZED` に保存

### 2. 取得漏れ・除外理由の可視化
- 除外データを削除せず、判定列と理由列を付与して保持
- `取得状況サマリー`、`除外理由別一覧`、`ジャンル不一致一覧`、`住所未取得一覧`、`詳細取得失敗一覧` を作成
- Googleマップ拡張機能の取得失敗やジャンル不一致をGAS上で確認可能

### 3. 重複排除機能
- 複数媒体で重複している店舗を以下のルールに基づき自動判定・統合
  - 電話番号が一致：重複確定
  - 店名＋住所が一致：重複確定
- 重複行は削除せず、`重複` タブに出力

### 4. チェーン店・施設判定
- リスト作成の基準リストである `MASTER_CHAIN` シートと突合し、大手チェーン店を自動除外
- 商業施設、ホテル、病院、学校などは `ビル管理除外` に分類
- 小規模ビルや階数表記などは完全除外せず `確認対象` に分類

### 5. タブ分け
- `営業対象 / 確認対象 / 重複 / チェーン店除外 / ビル管理除外 / エリア外 / 住所未取得 / 電話番号なし / ジャンル確認 / 取得失敗` に分類
- `検索ジャンル = カフェ` の場合、店名や取得元ジャンルのカフェ系キーワードから `カフェ` に補正

### 6. コムデスク投入用CSV生成機能
- コムデスクのインポート形式に合わせたヘッダーに自動整形
- 営業対象、ユニーク、単独店、施設対象、エリア内、電話番号あり、住所あり、HD対象ジャンルのみ出力
- `コムデスク投入用` タブに出力

## 一括処理フロー

`executeAllProcesses()` は以下を順に実行します。

1. `importCSVFiles()`
2. `executeNormalizeAndValidate()`
3. `executeDuplicateCheck()`
4. `executeChainCheck()`
5. `executeFacilityCheck()`
6. `executeSplitSheets()`
7. `executeGenerateComdeskCsv()`
8. `executeProcessLogSummary()`
