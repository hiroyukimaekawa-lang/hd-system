# 楽天トラベル宿泊施設 GAS（code-rakuten.js）

楽天トラベルをOctoparseでスクレイピングした宿泊施設データを取り込み、
チェーン・大規模施設を除外して「電気切り替え条件でHP無料作成」の営業に向いた
小規模宿泊施設（民宿・ペンション・ゲストハウス・グランピング等）だけを
営業リスト化するGAS。`gas/list-normalizer/code-restaurant.js`（飲食店側）と
同じ思想・命名規則で作られている。

## 主な除外ロジック（Ver1.3.0〜）

1. **部屋数（客室数）による除外**：15室以上の施設は、すでに一定規模の
   マーケティング体制・自社サイトを持っている可能性が高いとみなし除外。
   スクレイピング元CSVの「総部屋数」ラベル列の直後の値列から取得する
   （列番号ではなくラベル文字列で検索するため、Octoparseの列順が
   多少ズレても追従できる。`rktFindFieldByLabel_`）。
2. **チェーン除外キーワード**：`RAKUTEN_EXCLUDE_KEYWORDS`にアパホテル・
   ルートイン・東横INN等の全国チェーンに加えて、「ホテルグループ」という
   自己申告的な表記自体も除外条件にしている（BBHホテルグループ等、
   個別ブランド名を覚えなくても同種の中小チェーンを拾える）。
3. **営業対象／要確認キーワード**：民宿・ペンション・ゲストハウス・
   グランピング等は`RAKUTEN_TARGET_KEYWORDS`で営業対象に、
   旅館・ヴィラ・貸別荘等の判断が分かれるものは`RAKUTEN_CONFIRM_KEYWORDS`
   で要確認扱いにする。どちらにも一致しないものも要確認（安全側）。

## 対応しているCSV形式（自動判定）

- **一体型CSV**（Octoparse1タスクで住所・電話番号まで取得。実運用はこちら）：
  「楽天_raw」シートに`フィールド3`（住所）・`フィールド5`（電話番号）相当の
  値が入っていれば自動でこの方式として処理する。
- **二段階結合**（一覧CSV＋詳細CSVを別々に取得して突き合わせる旧方式）：
  「楽天_詳細抽出」シートにデータがあればこちらを優先する。

実データ確認済み：茨城県宿泊.csv（Octoparse出力、48列、`フィールド1`〜`フィールド7`が
名前/URL・住所ラベル/値・TELラベル/値・総部屋数ラベル/値のペアになっている形式）。
このCSVにはFAXの列が存在しないため、FAX欄は空欄になる（想定通りの挙動）。

## 使い方

1. スプレッドシートのメニュー「🏕 楽天トラベル宿泊施設」→「0. 初期タブ・投入フォルダを作成」
2. Octoparseで取得したCSVを「楽天_raw_CSV投入フォルダ」に入れる
3. 「🚀 全自動処理：投入済みCSVからCSV出力まで実行」を実行
4. `04_SALES_宿泊施設` / 地域別タブ / `完成版CSVエクスポート`フォルダのCSVを確認

## 部屋数のしきい値を変える場合

`RAKUTEN_ROOM_COUNT_EXCLUDE_THRESHOLD`（既定15）を変更する。
