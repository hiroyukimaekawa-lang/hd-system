# 楽天トラベル 宿泊施設スクレイパー（scrape.js）

Octoparseで手動実行していた「都道府県を指定して宿泊施設一覧＋住所・電話番号・
部屋数を取得する」作業を、都道府県名だけ指定すれば完結するNode.js製CLI。
Octoparseは使わない（このスクリプト自体が楽天トラベルのページを直接読みに行く）。

出力するCSVは `../code-rakuten.js`（GASの正規化・分類ロジック）がすでに
認識している列名（店名・住所・電話番号・FAX・総部屋数・都道府県 等）で
そのまま出力するため、GAS側の改修は不要。

## 実行方法

```bash
cd gas/rakuten-lodging/scraper
npm install
node scrape.js --pref 茨城県 --out ./output
```

初回は必ず `--max` で少数だけ試すことを推奨する。

```bash
node scrape.js --pref 茨城県 --max 10 --out ./output
```

### オプション一覧

| オプション | 説明 | 既定値 |
| --- | --- | --- |
| `--pref` | 都道府県名（必須。例: 茨城県） | - |
| `--out` | 出力先ディレクトリ | `./output` |
| `--hyoji` | 1ページあたりの取得件数 | 30 |
| `--delay` | リクエスト間隔(ms)。サイトへの配慮のため間隔を空けている | 500 |
| `--concurrency` | 詳細ページ取得の同時実行数 | 3 |
| `--max` | 取得件数の上限（デバッグ用） | 無制限 |

## ★この一次実装について（重要）

一覧ページ（`web.travel.rakuten.co.jp/portal/my/search_undecided.main`）と
詳細ページ（`travel.rakuten.co.jp/HOTEL/{id}/{id}_std.html`）の実データは、
茨城県のページを実際に1回ずつ取得して中身を確認した上でパーサーを書いている
（住所「〒312-0023茨城県ひたちなか市大平1-22-1」、TEL「029-273-7711」、
FAX「029-273-7742」、総部屋数「100室」等はすべて実データで確認済み）。

ただし、確認に使ったツールがHTMLをテキスト化して返す仕様だったため、
**実際のHTMLタグ構造（class名など）までは見れていない**。そのため抽出は
CSS/DOMセレクタに頼らず、「◯◯というラベル文字列の直後にある値」という
正規表現ベースにしてある（サイト側のクラス名変更に強い一方、想定外の
レイアウトの施設では取りこぼす可能性がある）。

また、このサンドボックス環境はネットワークがアローリスト制限されており、
`travel.rakuten.co.jp` への実際のHTTPリクエストを最後まで通しで実行して
確認することができなかった（`npm install`は通ったが、実際のスクレイピング
リクエストは`Connection blocked by network allowlist`でブロックされた）。
パース処理そのものは、実データから採取した文字列を使ったユニットテストで
正しく値が取れることを確認済みだが、**実際のサイトに対する動作は未検証**。

→ 初回実行は必ずあなたの手元（Claude Codeのターミナル、または通常のターミナル）
で `--max 10` 程度の少数から試し、出力CSVの中身を確認してほしい。
うまく取れていない列があれば、そのCSVとエラーメッセージを見せてもらえれば
抽出ロジックを調整する。

## 都道府県コード（f_chu）について

楽天トラベルの検索フォームは都道府県ごとに内部コード（f_chu）を使う。
「茨城県」は `ibaragi`（一般的なローマ字"ibaraki"ではない）であることを
実データで確認済み。他の46都道府県は `scrape.js` 内の `PREF_F_CHU` に
標準的なローマ字表記を仮で入れているが未検証。

実行して「検索結果が0件、または件数を読み取れませんでした」と出た場合は
コードが間違っている。以下の手順で正しい値を調べて `PREF_F_CHU` を修正する。

1. ブラウザで `https://travel.rakuten.co.jp/yado/` から対象の都道府県を開く
2. 表示されたページのURL、またはページ内の「絞り込み検索」等のリンクに
   `f_chu=xxxxx` という形でコードが含まれているので、その値を控える
3. `scrape.js` の `PREF_F_CHU` オブジェクトにその都道府県のコードを追記/修正する

## 出力したCSVをGASに投入する方法（2通り）

### 方法A: Google Drive for Desktopで同期する（一番簡単）

パソコンにGoogle Drive for Desktopを入れていれば、Driveの
「楽天_CSV投入フォルダ」はローカルのフォルダとして見えているはず。
`--out` にそのフォルダのパスを直接指定すれば、保存した瞬間にDriveへ
同期され、そのままGAS側の処理対象になる。

```bash
node scrape.js --pref 茨城県 --out "/Users/xxx/Google ドライブ/.../楽天_CSV投入フォルダ"
```

### 方法B: 手動でDriveにアップロードする

`--out ./output` のまま実行し、生成されたCSVをブラウザから
「楽天_CSV投入フォルダ」にドラッグ＆ドロップする。

## 完全自動化するには（GAS側のトリガー設定）

CSVを投入フォルダに置くだけで、スプレッドシートを開かなくても自動で
処理が走るようにするには、Google Apps Scriptのエディタで一度だけ
メニューから「⏰ 自動実行トリガーを設定（15分おき）」を実行しておく
（`code-rakuten.js` に追加済み）。これで15分ごとに投入フォルダを
チェックし、CSVがあれば自動で正規化・分類・営業リストCSV出力まで行う。

これで運用フローは：

1. `node scrape.js --pref 茨城県 --out <Drive同期フォルダ>` を実行（あなたの手元、またはClaude経由）
2. 最大15分待つ
3. 「04_SALES_宿泊施設」タブと「完成版CSVエクスポート」フォルダに結果が出る

という、都道府県名を指定するだけの流れになる。

## 既知の未対応・今後の改善余地

- 都道府県コード(f_chu)は茨城県以外は未検証（上記手順で随時修正が必要）。
- 楽天トラベル側のページ構造が変わった場合はパーサーの修正が必要（Octoparseを
  使っていた場合も同じ課題があったため、これは移行によるデメリットではない）。
- Google DriveへのCSVアップロードを完全に自動化する場合は、Google Drive APIの
  サービスアカウントを使う方法もあるが、認証情報のセットアップが必要になるため、
  まずはDrive for Desktop同期での運用を推奨している。必要であれば実装する。
