# HD事業部専用 機能一覧LP / 業務ツールポータル 要件定義

## 目的
HD事業部で日々の業務効率化・リスト作成のために稼働している複数のChrome拡張機能、GAS、および在庫管理等の分析機能を一箇所に統合し、誰でもすぐに「目的・使い方・リンク・ステータス」を確認できる**業務ツールポータル**として機能させる。

---

## サイト概要
- **仮タイトル**: HD事業部 業務ツールポータル
- **サブタイトル**: 営業リスト作成・店舗データ取得・架電管理を一元化するHD事業部専用ツール一覧
- **目的**: 
  1. 現在使える機能とステータス（運用中・開発中・改善中・検証中・構想中・停止中）の可視化
  2. 非エンジニアでも各ツールの目的・入力/出力・使い方を直感的に理解できる状態の確立
  3. 各種マニュアル、Googleスプレッドシート、Google Drive、GitHubへの一発アクセス

---

## ページ構造 (`index.html`)

### 1. ヘッダー
- ポータルのロゴマーク
- 登録ツール数 / 運用中 / 改善中 / 開発中の件数をリアルタイムで上部にバッジ表示

### 2. ファーストビュー (Hero)
- HD事業部ポータルの導入文
- 登録ツールの総数とステータス別ツールの集計ダッシュボード（件数表示）

### 3. コントロール・フィルターエリア
- **検索機能**: ツール名、カテゴリ、概要、キーワード（例: 「Googleマップ」「GAS」など）の部分一致検索
- **カテゴリフィルター**: 
  - すべて / データ取得 / リスト整形・統合 / コムデスク投入 / 架電管理 / アポ・訪問最適化 / 分析・レポート / マニュアル・運用ルール
- **ステータスフィルター**:
  - すべて / 運用中 / 改善中 / 開発中 / 検証中 / 構想中 / 停止中

### 4. 機能カードグリッド (`cards-grid`)
各ツールをカード形式で並列表示。
- **カードの表示内容**:
  - ツール名、カテゴリ
  - ステータスバッジ（運用中＝緑、改善中＝黄、開発中＝青など）
  - ツールの簡易概要
  - 最終更新日
  - 関連リンクアイコン（スプレッドシート、GitHub、マニュアル等がある場合のみ自動でミニアイコンとして表示）
  - 「詳細を見る」ボタン（クリックするとモーダルウィンドウで展開）

### 5. 詳細モーダル
カードクリック時に浮き上がるオーバーレイ。
- **表示内容**:
  - ツール名、ステータス、最終更新日
  - ツールの詳細概要
  - 解決する課題（手作業の工数削減など）
  - ターゲットユーザー / 使用頻度
  - 用途・目的
  - 入力データ・成果物（リスト形式）
  - 操作手順（1, 2, 3 ... のステップ形式）
  - 使用上の注意（APIキーの保護、規約関連など）
  - ロードマップ（今後予定している追加機能や改修）
  - 関連リンクボタン（設定されている項目のみ、リンクボタンとして自動生成）

### 6. システム更新・改善履歴
- 日付 / 更新したツール名 / 更新種別（新規追加、改善、バグ修正、運用変更、停止） / 変更内容の詳細説明
- 最初は5件のみ表示し、「過去の更新履歴を表示」をクリックすると全件展開

---

## データ管理方法 (`features.json`, `updates.json`)
HTMLに直接コンテンツをハードコードせず、ツール情報は `features.json`、更新履歴は `updates.json` で管理。
これにより、新規ツール追加時にはJSONファイルを1件追加するだけで、ポータル全体の件数サマリーやカード表示が自動的に同期・更新される設計とする。

## 開発環境・公開
- **フロントエンド**: Vanilla HTML, CSS, JavaScript (Next.jsやTailwindなどの外部重厚フレームワークはあえて使用せず、表示速度とファイルポータビリティを最優先)
- **ホスティング**: GitHub Pagesでの簡単な公開を想定
